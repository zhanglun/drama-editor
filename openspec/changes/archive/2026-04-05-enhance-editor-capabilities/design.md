## Context

### 当前状态
- 编辑器基于 TipTap 构建，已实现基础剧本格式（场景、对白、动作、转场）
- 使用 TipTap StarterKit 提供基础的富文本功能（粗体、斜体）
- 已有自定义节点视图（SceneNodeView、DialogueNodeView 等），但功能较基础
- 已有斜杠命令菜单，但只包含剧本专用元素

### 技术栈
- TipTap 2.x（基于 ProseMirror）
- React 18
- Tailwind CSS
- 自定义节点扩展模式已建立

### 约束
- 必须保持与现有剧本数据的向后兼容
- 不能破坏现有的剧本格式化功能
- 性能要求：大型剧本（100+ 场景）仍需流畅编辑
- 用户学习成本：新功能应直观易用

## Goals / Non-Goals

**Goals:**
1. 添加 Notion 风格的多级标题（H1-H6）、引用块、列表等通用文本格式
2. 增强自定义节点视图的视觉设计和交互体验
3. 扩展斜杠命令菜单，支持新的文本格式
4. 保持代码可维护性，遵循现有的扩展模式

**Non-Goals:**
1. 不实现协同编辑功能
2. 不实现表格、图片上传等高级功能
3. 不重构整个编辑器架构
4. 不改变现有的数据存储格式（仅扩展）
5. 不实现移动端适配（本次仅关注桌面端）

## Decisions

### 1. TipTap 扩展策略

**决策**: 使用官方 TipTap 扩展 + 自定义扩展组合

**理由**:
- 官方扩展（heading、blockquote、lists 等）经过充分测试，功能稳定
- 避免重复造轮子，降低维护成本
- 自定义扩展继续用于剧本专用节点（场景、对白等）

**替代方案考虑**:
- ❌ 全部自定义：工作量巨大，稳定性风险高
- ❌ 使用第三方扩展包：可能引入额外依赖，控制力较弱

**实现**:
```typescript
// ScriptEditor.tsx
extensions: [
  StarterKit.configure({
    heading: false, // 禁用 StarterKit 的 heading，使用独立的
  }),
  Heading, // 独立的 heading 扩展，可配置级别
  Blockquote,
  BulletList,
  OrderedList,
  CodeBlock,
  HorizontalRule,
  // 现有扩展
  Scene,
  Dialogue,
  Action,
  Transition,
  SlashCommand,
]
```

### 2. 自定义节点视图增强方案

**决策**: 采用组件化设计 + Tailwind CSS 样式系统

**理由**:
- 保持与现有项目技术栈一致
- Tailwind CSS 便于快速迭代和主题定制
- 组件化设计便于测试和维护

**设计原则**:
- 视觉层次：通过颜色、边框、间距区分不同节点类型
- 交互反馈：hover、选中状态清晰可见
- 一致性：所有节点遵循统一的设计语言

**实现策略**:
```
节点视图结构：
├── 节点容器（NodeViewWrapper）
│   ├── 节点类型标识（标签）
│   ├── 节点内容（可编辑区域）
│   └── 节点工具栏（hover 时显示）
```

### 3. 斜杠命令菜单扩展

**决策**: 扩展现有 SlashCommand，分组显示命令

**理由**:
- 复用现有代码，降低改动范围
- 分组设计使菜单更有组织性

**命令分组**:
```
剧本元素组：
- 场景、对白、动作、转场

文本格式组：
- 标题 1-6、引用、列表、代码块、分割线

未来扩展组：
- 可添加更多自定义元素
```

**实现**:
```typescript
const defaultItems: SlashMenuItem[] = [
  // 剧本元素
  { title: '场景', description: '插入场景标题', command: 'insertScene', icon: '🎬', group: 'script' },
  { title: '对白', description: '插入角色对白', command: 'insertDialogue', icon: '💬', group: 'script' },
  // ... 其他剧本元素
  
  // 文本格式
  { title: '标题 1', description: '大标题', command: 'setHeading1', icon: 'H1', group: 'format' },
  { title: '标题 2', description: '中标题', command: 'setHeading2', icon: 'H2', group: 'format' },
  // ... 其他格式
]
```

### 4. 数据格式兼容性

**决策**: 扩展现有 JSON 格式，不破坏旧数据

**理由**:
- 现有用户数据不能丢失或损坏
- TipTap 本身支持多节点类型共存

**策略**:
- 新节点类型（heading、blockquote 等）作为可选内容
- 旧剧本加载时，不存在的节点类型会被忽略（正常降级）
- 保存时，仅保存实际使用的节点类型

## Risks / Trade-offs

### Risk 1: 性能影响
**风险**: 添加更多扩展可能影响编辑器性能，特别是在大型剧本中

**缓解措施**:
- 使用 TipTap 的按需渲染机制
- 对大型文档实现虚拟滚动（如需要）
- 性能基准测试：确保 100+ 场景的剧本编辑流畅

### Risk 2: 学习成本
**风险**: 新功能可能增加用户学习成本

**缓解措施**:
- 保持斜杠命令菜单简洁直观
- 提供键盘快捷键提示
- 未来可添加新手引导

### Risk 3: 样式冲突
**风险**: 新的样式可能与现有样式冲突

**缓解措施**:
- 使用 CSS Modules 或 Tailwind 的作用域类名
- 充分测试新旧节点共存的场景
- 建立样式规范文档

### Trade-off: 功能完整性 vs 复杂度
**权衡**: 添加更多文本格式功能会增加代码复杂度

**决策**:
- 优先实现最常用的格式（标题、引用、列表）
- 其他高级格式（表格、图片）留待后续迭代
- 保持扩展点，便于未来添加

## Migration Plan

### 阶段 1: 添加新扩展（无破坏性）
1. 安装新的 TipTap 扩展依赖
2. 在 ScriptEditor 中注册新扩展
3. 扩展斜杠命令菜单
4. 测试新功能

**回滚策略**: 移除新扩展注册，不影响现有功能

### 阶段 2: 增强自定义节点视图
1. 逐个更新节点视图组件（Scene、Dialogue、Action、Transition）
2. 添加节点工具栏
3. 优化样式

**回滚策略**: 保留旧视图组件作为备份，可快速切换

### 阶段 3: 测试与优化
1. 全面功能测试
2. 性能测试（大型剧本）
3. 浏览器兼容性测试
4. 用户反馈收集

**验证标准**:
- ✅ 旧剧本正常加载和编辑
- ✅ 新功能正常工作
- ✅ 100+ 场景剧本编辑流畅（无明显卡顿）
- ✅ 主流浏览器（Chrome、Firefox、Safari）兼容
