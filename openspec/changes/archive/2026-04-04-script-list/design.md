## Context

用户需要在剧本列表页面查看和管理剧本。项目已有 `ScriptList` 和 `ScriptCard` 组件，但它们功能已完整实现，但未被集成到 `App.tsx` 的 `ScriptsPage` 跴。此外，还需要将现有的 `ScriptList` 组件集成到 `ScriptsPage` 中，还需要将删除功能从物理删除改为逻辑删除（软删除）。



## Goals / Non-Goals

### Goals
- 实现剧本列表页面的完整功能展示
- 支持点击卡片进入剧本详情页
- 提供删除功能，支持二次确认
- 删除时执行逻辑删除，不显示已删除的剧本

- 卡片展示剧本的标题、创建时间、最后修改时间

- 支持搜索和排序功能（已存在于 ScriptList 组件中）

- 保持加载状态和错误处理

- 巻加空状态处理

- 支持响应式设计

### Non-Goals
- 不改变现有编辑器功能
- 不改变版本历史功能
- 不改变导出功能
- 不添加用户认证（后续需求)
- 不添加实时协作功能（后续需求）

- 不实现分页功能（当前数据量预计较小，使用前端过滤即可可)

## Decisions
1. **数据模型**: 在 `Script` 模型中添加 `deleted_at` 字段（`gorm.Model` 类型），支持逻辑删除
2. **API 设计**: 保持现有 RESTful API 不变，删除操作通过 PATCH 请求将 `deleted_at` 设置为当前时间
3. **UI 组件**: 复用现有的 `ScriptCard` 组件，调整显示字段以展示创建时间
4. **搜索排序**: 继续使用前端搜索和排序，避免后端压力
5. **逻辑删除策略**: 使用 Gorm 的软删除功能（`gorm.DeletedAt` 字段），这是 Gorm 的最佳实践
## Risks / trade-offs

| 兼容性 | 保持与现有组件 API 兼容 |
| 过渡成本 | 无需数据迁移 |
| 安全风险 | 低 - 仅涉及用户数据，无安全风险 |
